# Facilitator Guide — Run of Show

For the person(s) running the room + livestream.

## Framing (open)

- One line: "Bring your laptop, leave with a stack."
- Show `docs/STACK-MAP.md` on screen. Name the 10 stations. Set the promise: by the end,
  a visible system with receipts.
- Confirm the room is green on `docs/ATTENDEE-SETUP.md`.

## Flow

Work the stations in order. Each station:
1. **Demo** the finished `solution/` for ~2 minutes (what "done" looks like).
2. **Build** from `starter/` together.
3. **Checkpoint** — everyone runs the station's checkpoint test; nobody left behind.

Stations are dependency-ordered but self-contained, so if a station runs long you can
hand out its `solution/` and keep the room synced.

## Livestream notes

- Keep the STACK-MAP visible between stations as the "where are we" anchor.
- Each station's receipts/telemetry output is good on-camera material — it's the visible
  proof the system works.
- Drop the repo + Luma link in chat at every station transition.

## Close

- Run Station 10's dashboard live — the whole stack, visible, with receipts.
- CTA: clone the repo, star it, and the follow-up (newsletter / recap).
