Upstream · Observe

Observe —
every event, as it happens.

Each agent emits a structured event — a run id, the agent name, what it did, a score. They land in a Cloudflare Queue so a slow agent never blocks the board. Capture first, reason later.

Structured events

run_id · agent · kind · score · ts — one shape for everything, from a harness tick to a full run result.

Queue ingest

Events publish to observability-events. The consumer drains batches into D1 without back-pressure on the emitter.

Cheap to emit

A fetch to the Worker is the whole SDK. No agent needs to know the schema of the board.

Replayable

Events are the log. Rebuild runs, agents, and the leaderboard from the event stream at any time.

Cloudflare Queues structured events batch consumer replay-safe

The Observed · Platform

ClaudeFlare —
four bets on what infra should be.

A DuraClaw fork that bets hard on what DuraClaw doesn't prioritize — and every one of these Workers reports into the board you just saw.

Bet 1 · Ephemeral Workers

Every task is a Worker that exists only long enough to complete its job. No warm processes. No idle billing. No cleanup debt. A single-use function, not a service.

Bet 2 · Auto-docs on Workers Assets

Every layer ships a flair.layer.json. A doc-gen Worker reads it, calls Claude, generates HTML, ships via wrangler deploy. Docs never go stale.

Bet 3 · ClawBox desktop control plane

Tauri v2 native client → OpenClaw Gateway via WebSocket RPC. Launch · tail · schedule · sessions. Local-first — no dashboard round-trips.

Bet 4 · Agent Client Protocol

JSON-RPC — the LSP-for-coding-agents. Zed · VSCode · acpx · ClawBox all drive Workers through session/new · prompt · cancel. One mental model, desktop to edge.

TypeScript · pnpm open source · MIT DuraClaw fork Tauri v2

Input → transform → output → exit.

One task per Worker

Multi-step work is chained Workers via Queue. Never a Worker that polls or loops.

Stateless by default

All state lives in KV / D1 / R2 — restart anything, lose nothing.

ULID + idempotent

Re-running the same ULID gives the same output. Safe to retry, safe to replay.

The Observed · Each layer owns one job

Queue owns delivery.
DO owns state.

The same separation the observability loop uses — async ingest, a single writer for truth, purpose-built stores — keeps fifty concurrent conversations from stepping on each other.

Queue owns delivery

Meta needs an ACK in <5s — an LLM call would never fit. The Queue absorbs the latency and retries the misses.

DO owns state

idFromName(user_id) pins one Durable Object per phone. Single writer per user, no global locks.

D1 owns records

messages · prompt_logs · failed_messages · usage_summary — batch INSERT, queryable truth.

R2 owns exports

Breakaway zip on !export. Signed URL, 7-day TTL — the user walks away with their data.
